تفاوت فریمورک و کتابخانه در چیست؟

کتابخانه فقط مجموعه ای از api ها(مثله کلاس ها,متدها,ماژول ها,...) هست که اونا رو به کدخودمون اضافه میکنیم, و برای ساده تر شدن کد نویسی از اونا استفاده میکنیم مثال خیلی ساده اون کتابخانه ی Math توی زبان C هست که توی اکثر زبان های برنامه نویسی هم وجود داره, مثلا اگه بخواهیم لگاریتم ی عدد رو بدست بیاریم دیگه نیازی به دونستن الگوریتم بدست آوردن اون نیست خیلی ساده از تابع کلاس Math استفاده میکنیم.

فریم ورک ی مفهوم پیچیده تره که درواقع به این صورته که وقتی API ها در یک مجموعه واحد برای یک هدف مشخصی قرار میگیرن و الگو های خاصی رو پیروی کنن (مثله مدل MVC) دیگه فراتر از API هستن و بهش میگن فریم ورک , درواقع فریم ورک مثله یک چارچوب واستخوان هست که نیاز هست برنامه نویس با نوشتن کد اونو کامل کنه و این کدها همان گوشت برای استخوان(فریم ورک) هست به طور کلی فریم ورک یک چارچوب به حساب می‌آید که معمولاً برنامه نویس‌ها برای توسعه و طراحی نرم افزار از آن استفاده می‌کنند. استفاده از فریم ورک به ساده شدن توسعه پردازش‌ها کمک می‌کند. یکی از دلیل‌های این سادگی این موضوع است که فریم ورک از نوشتن مکرر کدهای مشابه جلوگیری خواهد کرد. همچنین باعث ایجاد شیوه‌های خوب و مناسب و سازگاری در کدها می‌شود. بنابراین می‌توان گفت که فریم ورک، مجموعه‌ای از ابزارها و ماژول‌های گوناگون است که می‌توانند در پروژه‌های مختلف مورد استفاده قرار بگیرند.

چرا ری اکت کتابخانه محسوب می شود؟

React برای ساخت برنامه های وب تمام عیار ساخته شده است. در نتیجه، اغلب با ابزارهای دیگری که توسعه‌دهندگان برای ساخت برنامه‌ها استفاده می‌کنند، مانند Angular، Vue و Svelte مقایسه می‌شود. React با جاوا اسکریپت نوشته شده است و برای ساخت برنامه های بهتر جاوا اسکریپت استفاده می شود. ما به طور خاص به React به عنوان یک کتابخانه اشاره می کنیم. اما چه چیزی React را به یک کتابخانه و نه یک چارچوب تبدیل می کند؟ دلیل آن زمانی مشخص می شود که به ابزارهای مشابه دیگری که برای ایجاد برنامه های وب کامل استفاده می شوند نگاه کنیم. بیایید به پروژه ای مانند Angular نگاه کنیم که هدفی مشابه با React دارد (ایجاد برنامه های وب تک صفحه ای). چیزی که آن را متمایز می کند این واقعیت است که وقتی یک پروژه Angular را راه اندازی می کنید، تقریباً با تمام مواردی که برای ایجاد یک برنامه کامل و در مقیاس بزرگ به آن نیاز دارید، بوت استرپ می شود. بسیاری از توسعه‌دهندگان دوست دارند به چارچوب‌ها یا راه‌حل‌های مشابه به عنوان «باتری‌ها» اشاره کنند. فریم‌ورک‌ها برای شرکت‌ها و هر کسی که به دنبال ساخت برنامه‌های جاوا اسکریپت سازمانی است، انتخاب رایجی است، زیرا شامل منابعی است که احتمالاً یک برنامه در مقیاس بزرگ به آن نیاز دارد. این شامل ابزارهای داخلی برای کارهای رایج مانند ایجاد فرم ها، اجرای آزمایش های خودکار، ایجاد درخواست های شبکه و موارد دیگر است. به طور خلاصه، هر چیزی که برای ایجاد یک برنامه کامل نیاز دارید، در پروژه Angular شما در هنگام تولید گنجانده می شود. اما در مورد React اینطور نیست. این بدان معناست که برای ساخت برنامه‌های React کامل، باید این بسته‌ها و ابزارها را خودتان انتخاب کنید.
